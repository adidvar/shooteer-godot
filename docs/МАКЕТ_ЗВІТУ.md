## Слайд 14 — Реалізація управління

**Система руху (MovementComponent.cs):**
```
ProcessMovement() кожен _PhysicsProcess()
  ├── IsOnFloor() → GroundMove()
  │     ├── ApplyFriction() — сповільнення при відпусканні
  │     ├── Accelerate() — прискорення у напрямку
  │     └── Jump при _wishJump = true
  └── !IsOnFloor() → AirMove()
        ├── AirAccel = 2.0 (менш ніж GroundAccel)
        └── Гравітація: velocity.Y -= gravity * delta
```

**Огляд мишею (Player.cs):**
- `Input.GetLastMouseVelocity()` → повертає тіло по Y, камеру по X
- `GlobalSettings.MouseSensitivity` застосовується до обох осей

**Управління зброєю (WeaponController.cs):**
- `Input.IsActionPressed("shoot")` → `activeWeapon.Fire()`
- Перемикання: клавіші 1–4 або колесо миші
- WeaponSwitched сигнал → оновлення HUD

## Слайд 15 — Реалізовані взаємодії

### Взаємодія 1: Зброя → Гравець/Об'єкт (тип 2 — небезпечний об'єкт)
- **Механізм:** `RayCast3D` (BasicWeapon) або фізичне зіткнення (Rocket)
- **Перевірка типу:** `collider is Player` → RPC збитки; `collider.HasNode("HealthComponent")` → пряма шкода
- **Реакція:** зменшення HP, HitMarker на HUD, смерть → _isDead = true, відродження через 3с
- **Захист від повторних спрацювань:** `_cooldown > 0 → return`

### Взаємодія 2: Гравець → Аптечка (тип 3 — колекційний об'єкт) ← *Учасник 2*
- **Механізм:** `Area3D.BodyEntered` → перевірка `body is Player`
- **Реакція:** `HealthComponent.Heal(25)`, `QueueFree()` → Timer → відновлення
- **Захист від повторних спрацювань:** `_pickedUp` прапор, скидається при відновленні

### Взаємодія 3: Гравець → JumpPad (тип 4 — тригерна зона) ← *Учасник 2*
- **Механізм:** `Area3D.BodyEntered` → `body.Velocity = new Vector3(…, 15, …)`
- **Реакція:** різкий стрибок, частинки
- **Захист від повторних спрацювань:** `_cooldownTimer > 0 → return` (0.5с між активаціями)

### Взаємодія 4: Гравець → HazardZone (тип 2 — небезпечний об'єкт) ← *Учасник 3*
- **Механізм:** `Area3D.BodyEntered/Exited` → список тіл всередині → Timer тік
- **Реакція:** -5 HP кожні 0.5с поки в зоні, пульсуюча підсвітка зони
- **Захист від повторних спрацювань:** перевірка `_bodiesInside.Contains(body)` перед шкодою

### Взаємодія 5: Гравець → DestructibleCrate (тип 1 — перешкода) ← *Учасник 3*
- **Механізм:** `HealthComponent.TakeDamage()` при влученні; зменшення HP → RigidBody імпульс
- **Реакція:** ящик відлітає, при 0 HP → ефект частинок + звук + `QueueFree()`
- **Захист від повторних спрацювань:** `_isDestroyed` прапор

## Слайд 16 — Схема умов перемоги/поразки

```
         Початок раунду
               │
               ▼
    ┌─── Гравець живий ───┐
    │                     │
    ▼                     ▼
Отримує шкоду     Знищує ворога/бота
    │                     │
    ▼                     ▼
HP = 0 ?            Kills++
    │                     │
   Так              Kills >= FragLimit ?
    │                     │
    ▼                    Так
  Смерть                  │
  +1 Death                ▼
  Відродження      ПЕРЕМОГА
  через 3с         ─────────────────
    │               Показати таблицю
    │               результатів
    │               Кнопка "Меню"
    │
    ▼
  Таймер раунду = 0 ?
    │
   Так
    │
    ▼
  Перемагає гравець
  з найбільшою кількістю кілів
```

**Умова перемоги:** перший гравець набирає `FragLimit` кілів (за замовчуванням 10)  
**Альтернативна умова:** завершення таймера раунду (наприклад, 5 хвилин)  
**Умова поразки:** для конкретного гравця — набрати найменше кілів  
**Перехід:** після екрану результатів → автоматично в MainMenu через 10 секунд
