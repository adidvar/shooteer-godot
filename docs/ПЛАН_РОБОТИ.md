## Зміст кожного розділу (орієнтир для тексту слайдів)

### Аналіз аналогів

**Аналог 1 — Quake Champions (Bethesda, 2017)**
- Жанр: Arena FPS; Платформи: PC (Steam)
- Мета: перша/найбільша кількість фрагів за раунд; особливості: hitscan-зброя, стрейф-джамп, rocket jump, аренні мапи
- Сильні: висока швидкість, кар'єра гравця, різноманітна зброя, e-sport потенціал
- Слабкі: висока планка входу, застаріла f2p-модель, мала аудиторія онлайн

**Аналог 2 — Splitgate (1047 Games, 2021)**
- Жанр: Arena FPS з порталами; Платформи: PC, PS4/5, Xbox
- Мета: фраги та командні режими; особливості: портална система Aperture-style, TDM/CTF
- Сильні: унікальна портальна механіка, безкоштовний, кросплей
- Слабкі: портали збивають з толку нових гравців, мала глибина поза порталами

**Аналог 3 — Unreal Tournament 3 (Epic Games, 2007)**
- Жанр: Arena FPS; Платформи: PC, PS3
- Мета: Deathmatch/CTF/Onslaught; особливості: Translocator, класичний арсенал (Rocket Launcher, Shock Rifle, Link Gun)
- Сильні: різноманіття режимів, швидкий темп, велика кількість зброї
- Слабкі: застаріла графіка, маленька онлайн-спільнота, складна модифікація

### Базова ідея гри
- **Жанр:** Arena FPS Multiplayer | **Платформа:** PC (Windows/Linux)
- **Мета гри:** перший гравець, що досяг ліміту фрагів (FragLimit), перемагає
- **Сюжет/наратив:** 2077 рік. Мегакорпорації влаштовують кіберпанк-гладіаторські бої в неон-аренах для розваги VIP-клієнтів. Гравці — найманці-кіборги, що змагаються в "Неон Страйк".
- **Аудиторія:** 16–30 років, фанати Arena FPS (Quake, UT), гравці PC-шутерів

### Сетінг
- **Час:** 2077, нео-футуристичне майбутнє
- **Місце:** Закрита арена-куб у висотному неон-місті, голографічні рекламні щити, матеріал стін — метал + неон-підсвітка, кислотні пастки на підлозі
- **Візуальний стиль:** тональна палітра — чорний фон + неон (синій, малиновий, жовтий); стиль схожий на Tron Legacy + Mirror's Edge Catalyst; прості геометричні форми арени; Orbitron-шрифт в UI

### USP — унікальні особливості
1. **Арсенал із 4 принципово різних зброй** одночасно (Assault Rifle, Laser Sniper, Homing Rocket, Lightning Gun) — кожна із власною візуальною мовою
2. **Інтерактивна арена** — Jump Pads (стрибкові майданчики), Hazard Zones (кислотні зони), Destructible Crates (фізичні ящики зі здоров'ям)
3. **Кіберпанк-естетика** у мінімалістичному виконанні — неон UI, шрифт Orbitron, емісійні матеріали
4. **Реальний мультиплеєр** LAN/Online через ENet (не симуляція)

### Основні ігрові механіки
1. Рух гравця: WASD + мишка (погляд), стрибок, гравітація
2. Стрільба: 4 типи зброї — hitscan (Assault Rifle, Laser Sniper), projectile (Homing Rocket), continuous beam (Lightning Gun)
3. Система пошкоджень та відродження: HP → 0 → смерть → авто-респаун через 3с
4. Перемикання зброї: клавіші 1–4 або коліщатко миші
5. Умова перемоги: FragLimit (10 фрагів) → оголошення переможця

### Модулі/компоненти + взаємодія
```
Player (CharacterBody3D)
  ├── MovementComponent — рух та гравітація
  ├── HealthComponent — HP, сигнали Died/HealthChanged
  └── WeaponController — керує набором зброї
        ├── BasicWeapon (Assault Rifle) — hitscan
        ├── LaserSniper — hitscan+beam
        ├── HomingRocketWeapon → Rocket (CharacterBody3D) — projectile+AI
        └── LightningGun — cone beam damage

Map (Node3D)
  ├── Players (Node) ← MultiplayerSpawner
  ├── JumpPad (Area3D) — імпульс гравця вгору
  ├── HazardZone (Area3D) — тік-дамаг
  ├── HealthPickup (Area3D) — лікування
  └── DestructibleCrate (RigidBody3D+HealthComponent) — руйнівний об'єкт

Main (Node)
  ├── MainMenu (Control) — host/join/settings
  ├── HUD (CanvasLayer) — HP, зброя, hit marker, пауза
  └── LevelContainer (Node) ← Map завантажується сюди
```

### Пітч (до 300 слів)
**Neon Strike** — швидкісний мультиплеєрний шутер для PC в стилі кіберпанк, де 2–4 гравці б'ються на неон-арені до досягнення ліміту фрагів.

Гра розрахована на фанатів класичних arena FPS (Quake, Unreal Tournament), які хочуть швидкий, хаотичний і красивий шутер без мікротранзакцій і черг матчмейкінгу — просто LAN або пряме підключення за IP і бій починається.

Головна відмінність від конкурентів — 4 зброї з принципово різною логікою: автомат для загального бою, лазерна гвинтівка для снайперингу, самонавідна ракета для переслідування ворогів за укриттями, і блискавкова гармата, що б'є конусом і відштовхує від стін. Арена не пасивна: стрибкові майданчики дозволяють мити та неочікувані траєкторії, кислотні зони змушують постійно рухатись, ящики зі здоров'ям дають тактичний вибір — зруйнувати чи підібрати.

**Для кого:** гравці 16–30 років, PC, мультиплеєр з друзями.  
**Що нового:** поєднання кіберпанк-естетики з класичним arena FPS геймплеєм і 4-ма унікальними зброями в одній грі.  
**Чим приваблює:** яскравий неон-стиль, динамічний темп, простий вхід — запустив, підключився, пограв.

**Взаємодія Б — тип 4: ~25 рядків у `Map.cs` + ~20 рядків у `HUD.cs`.**

#### Завдання:

**1.1 Лічильник кілів у `Map.cs` (~25 рядків)**
- Додати поле `private Dictionary<long, int> _kills = new();` та `private const int FragLimit = 10;`

### Що потрібно для звіту
1. **Діаграма станів ракети:**

## Учасник 3 — Механіка Г: Блискавкова гармата (доопрацювання)

### Що вже є
`LightningGun.cs`:
- При `Fire()` і `Input.IsActionPressed("shoot")`: промінь (`RayCast3D` або `AimDetector`), конусні частинки, тік-дамаг кожні `TickRate` секунд
- Візуальний `_beamMesh` (CylinderMesh, синій неон), `_coneParticles`, `_beamParticles`

### Що потрібно доопрацювати (~20 рядків)
1. **Перевірити `Fire()` логіку** — у базовому класі `WeaponBase` `Fire()` викликається з `WeaponController._Process`. Для LightningGun потрібно щоб `_isFiring = true` при `IsActionPressed` і `_isFiring = false` при відпусканні.  
   Додати в `WeaponController._Process`:  
   ```csharp
   if (_activeWeapon is LightningGun lg)
       lg.SetFiring(Input.IsActionPressed("shoot"));
   else if (Input.IsActionPressed("shoot"))
       _activeWeapon?.Fire();
   ```
2. **Масштаб beam mesh** — при стрільбі `_beamMesh` має тягнутись від MuzzlePoint до точки попадання. Встановити `Scale.Y = відстань`, `Position = середина між точками`.
3. **Дамаг гравцям у конусі** — `_coneParticles.GlobalPosition = GlobalPosition` + query SpaceState для гравців у сфері + перевірка кута.

### Що потрібно для звіту
1. **Блок-схема роботи:**  
   ```
   IsActionPressed("shoot") == true → _isFiring = true
     → _tickTimer -= delta
     → _tickTimer <= 0: нанести дамаг усім у конусі → _tickTimer = TickRate
     → Показати _beamMesh (позиція + масштаб)
     → _coneParticles.Emitting = true
   IsActionPressed("shoot") == false → _isFiring = false
     → _beamMesh.Visible = false
     → _coneParticles.Emitting = false
   ```
2. Скріншоти: beam ефект під час стрільби, частинки конуса
3. Лістинг `LightningGun.cs`
